# 系统架构图

> 本文档用 Mermaid 描述 `stability_harness_loop_multiagent` 的架构，
> 对应 `AGENTS.md` 与三引擎设计。三引擎在**模块层**强制互不 import，
> 所有跨引擎通信只走 `EventBus`（pub/sub + request/reply + `#` 通配）。

## 1. 三引擎总览

`harness ←(EventBus)→ loop ←(EventBus)→ multi_agent`，边界在模块层强制。

```mermaid
flowchart LR
    subgraph H[Harness 引擎]
        H1[Runtime 运行时/监督]
        H2[bus EventBus 唯一接缝]
        H3[Watchdog 存活/死锁]
        H4[Telemetry 可观测]
        H5[Governance 鉴权/熔断/预算]
        H6[Verifier 护栏/校验]
    end
    subgraph L[Loop 引擎]
        L1[ControlLoop 控制循环]
        L2[DecisionAuthority 事实独裁]
        L3[SharedContext 不可变上下文]
        L4[TerminationPolicy 中止条件]
        L5[Scheduler 调度/退避]
    end
    subgraph M[MAS 多智能体]
        M1[Worker 执行]
        M2[Advisor 投票/事件]
        M3[Observer 记录/面板]
        M4[TargetAdapter 目标适配]
    end
    H2 <-->|EventBus| L1
    L1 <-->|EventBus| M1
    L1 <-->|EventBus| M2
    L1 <-->|EventBus| M3
    M1 -->|操作| M4
    H5 -.治理闸门.-> M1
    H6 -.校验.-> M1
```

## 2. 控制循环（ControlLoop）

`sense → plan → act → check → decide → halt` 的确定性循环，每轮必产生 `Verdict`，
且受 `max_rounds` / `max_duration` 硬上限保证一定终止。

```mermaid
flowchart TD
    Start([开始]) --> Tick{loop/tick}
    Tick --> Sense[感知: 收集 target/checked + target/recovered]
    Sense --> Plan[计划: 生成本轮操作]
    Plan --> Act[执行: Worker 经 TargetAdapter 操作目标]
    Act --> Check[探测: probe 产出事实]
    Check --> Decide{DecisionAuthority.decide}
    Decide -->|事实全 True| Pass[Verdict: pass]
    Decide -->|任一事实 False| Fail[Verdict: fail]
    Decide -->|error / 不确定| Warn[Verdict: warn 60 保守]
    Decide -->|critical 事件| Recheck[recheck x1]
    Pass --> Halt{TerminationPolicy}
    Fail --> Halt
    Warn --> Halt
    Recheck --> Sense
    Halt -->|未满足| Tick
    Halt -->|max_rounds/duration 或 abort| Done([loop/done 退出])
```

## 3. 投票与事实独裁

Advisor 只投 `(risk, confidence)`，不裁决；`DecisionAuthority` 拥有唯一裁决权。
任一事实为 `False` → 本轮 `fail`，不可被风险分或投票翻转为 `pass`。

```mermaid
flowchart LR
    A1[Advisor A] -->|vote risk,conf| Combine[combine_votes / 本地镜像合并]
    A2[Advisor B] -->|vote risk,conf| Combine
    A3[Advisor C] -->|vote risk,conf| Combine
    Facts[探测事实] --> Decide{DecisionAuthority}
    Combine -->|附加 warn/recheck 注解| Decide
    Decide -->|事实 False| F[FAIL 不可翻转]
    Decide -->|事实全 True| P[PASS]
    Decide -->|error| W[WARN 60 保守]
```

## 4. 数据驱动场景层（门禁对讲）

零代码新增用例：复制 YAML 模板改字段，`scenario_worker` 把探测结果产出事实交给 `DecisionAuthority`。

```mermaid
flowchart LR
    Y[cfg YAML<br/>target+stress+probe+loop] --> Runner[run_scenario]
    Runner --> Adapter[ScenarioAdapter<br/>真机/ dry-run]
    Adapter --> Worker[ScenarioWorker]
    Worker -->|act| Target[(门禁对讲设备)]
    Worker -->|probe 事实| Decide[DecisionAuthority]
    Decide --> Report[治理面板/Observer]
```

## 不变量速记

- **循环终止**：`max_rounds` / `max_duration` 硬上限，不可能死锁。
- **裁决产生**：每轮必出 `Verdict`。
- **事实独裁**：注入失败事实强制 `fail`，即使 Advisor 投低风险。
- **事件扇出**：Observer 必收 `loop/done`，总线端到端可用。
- **防死锁**：每个跨引擎 `await` 都有超时 + 确定性兜底；`recheck` 上限 1 次。
